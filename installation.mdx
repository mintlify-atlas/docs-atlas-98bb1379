---
title: "Installation"
description: "Install Dufs on macOS, Linux, or Windows using cargo, Homebrew, Docker, or a pre-built binary."
---

Dufs v0.45.0 is a single binary with no runtime dependencies. Pick the method that fits your workflow.

<Tabs>
  <Tab title="cargo">
    <Steps>
      <Step title="Install Rust">
        If you don't have Rust installed, get it from [rustup.rs](https://rustup.rs):

        ```bash
        curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
        ```
      </Step>
      <Step title="Install Dufs">
        ```bash
        cargo install dufs
        ```

        Cargo compiles Dufs from source and places the binary in `~/.cargo/bin/`. Make sure that directory is on your `PATH`.
      </Step>
      <Step title="Verify the installation">
        ```bash
        dufs --version
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="Homebrew">
    <Steps>
      <Step title="Install Dufs">
        ```bash
        brew install dufs
        ```
      </Step>
      <Step title="Verify the installation">
        ```bash
        dufs --version
        ```
      </Step>
    </Steps>

    <Note>Homebrew is available for macOS and Linux. Install it from [brew.sh](https://brew.sh) if you don't have it yet.</Note>
  </Tab>
  <Tab title="Docker">
    <Steps>
      <Step title="Pull and run">
        The official image is built from scratch and contains only the Dufs binary. Mount the directory you want to serve at `/data` inside the container:

        ```bash
        docker run -v `pwd`:/data -p 5000:5000 --rm sigoden/dufs /data -A
        ```

        This mounts your current directory, forwards port `5000`, and enables all operations with `-A`.
      </Step>
      <Step title="Open the UI">
        Navigate to [http://localhost:5000](http://localhost:5000) in your browser.
      </Step>
    </Steps>

    <Tip>Use `DUFS_*` environment variables with `-e` flags to configure Dufs without changing the command. See [Environment Variables](/configuration/environment-variables) for the full list.</Tip>
  </Tab>
  <Tab title="Binary download">
    <Steps>
      <Step title="Download the binary">
        Go to the [GitHub Releases page](https://github.com/sigoden/dufs/releases) and download the archive for your platform:

        | Platform | File |
        | -------- | ---- |
        | macOS (Apple Silicon) | `dufs-v*-aarch64-apple-darwin.tar.gz` |
        | macOS (Intel) | `dufs-v*-x86_64-apple-darwin.tar.gz` |
        | Linux (x86\_64) | `dufs-v*-x86_64-unknown-linux-musl.tar.gz` |
        | Linux (ARM64) | `dufs-v*-aarch64-unknown-linux-musl.tar.gz` |
        | Windows | `dufs-v*-x86_64-pc-windows-msvc.zip` |
      </Step>
      <Step title="Extract the archive">
        <CodeGroup>

        ```bash macOS / Linux
        tar -xzf dufs-*.tar.gz
        ```

        ```powershell Windows
        Expand-Archive dufs-*.zip
        ```

        </CodeGroup>
      </Step>
      <Step title="Add to PATH">
        Move the `dufs` binary to a directory on your `PATH` so you can run it from anywhere.

        <CodeGroup>

        ```bash macOS / Linux
        sudo mv dufs /usr/local/bin/
        ```

        ```powershell Windows
        # Move dufs.exe to a folder already on your PATH, or add its location to PATH via System Settings.
        Move-Item dufs.exe C:\Windows\System32\
        ```

        </CodeGroup>
      </Step>
      <Step title="Verify the installation">
        ```bash
        dufs --version
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Shell completion

Dufs can generate completion scripts for your shell. Run the command for your shell and follow the setup instructions below.

<Tabs>
  <Tab title="bash">
    ```bash
    dufs --completions bash > ~/.local/share/bash-completion/completions/dufs
    ```

    If the completions directory doesn't exist yet:

    ```bash
    mkdir -p ~/.local/share/bash-completion/completions
    dufs --completions bash > ~/.local/share/bash-completion/completions/dufs
    ```

    Then reload your shell:

    ```bash
    source ~/.bashrc
    ```
  </Tab>
  <Tab title="zsh">
    ```bash
    dufs --completions zsh > "${fpath[1]}/_dufs"
    ```

    Then rebuild the completion cache:

    ```bash
    rm -f ~/.zcompdump && compinit
    ```
  </Tab>
  <Tab title="fish">
    ```fish
    dufs --completions fish > ~/.config/fish/completions/dufs.fish
    ```

    Fish picks up completions automatically — no reload needed.
  </Tab>
  <Tab title="PowerShell">
    ```powershell
    dufs --completions powershell >> $PROFILE
    ```

    Reload your profile:

    ```powershell
    . $PROFILE
    ```
  </Tab>
  <Tab title="elvish">
    ```bash
    dufs --completions elvish >> ~/.config/elvish/rc.elv
    ```

    Restart Elvish or source the file to apply the completions.
  </Tab>
</Tabs>
