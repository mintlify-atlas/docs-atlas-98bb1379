This guide takes you from a bare installation to a fully functional file server. Each step builds on the previous one.

<Note>Make sure Dufs is installed before you start. See the [Installation](/installation) guide if you haven't done that yet.</Note>

## Serve your first directory

<Steps>
  <Step title="Start Dufs in the current directory">
    ```bash
    dufs
    ```

    Dufs starts in read-only mode by default. Visitors can browse and download files, but cannot upload or delete anything.
  </Step>
  <Step title="Open the web interface">
    Navigate to [http://localhost:5000](http://localhost:5000) in your browser. You'll see a directory listing for the folder you started Dufs in.

    <Tip>Dufs binds to `0.0.0.0:5000` by default, so it's reachable from other devices on your local network as well.</Tip>
  </Step>
</Steps>

## Enable all operations

To allow uploading, deleting, searching, archiving, and hashing, pass the `-A` flag:

```bash
dufs -A
```

If you only need specific operations, use the individual flags instead:

```bash
dufs --allow-upload --allow-delete
```

## Serve a specific path

Pass a directory or file path as the first argument:

```bash
dufs Downloads
```

To serve a single file:

```bash
dufs linux-distro.iso
```

## Add a password

Use `-a` to require authentication. The format is `user:pass@path:permission`:

```bash
dufs -a admin:123@/:rw
```

This makes `admin` with password `123` the only user who can read or write anything under `/`. Anonymous access is blocked entirely.

To allow public read-only access while restricting writes to authenticated users:

```bash
dufs -a admin:admin@/:rw -a guest:guest@/
```

<Note>Auth permissions are capped by the global permission flags. If `--allow-upload` is not set, no user — even one with `:rw` — can upload files.</Note>

## Use curl to upload and download files

Dufs has a full HTTP API that works with standard `curl` commands.

<Steps>
  <Step title="Start Dufs with uploads enabled">
    ```bash
    dufs -A
    ```
  </Step>
  <Step title="Upload a file">
    ```sh
    curl -T path-to-file http://127.0.0.1:5000/new-path/path-to-file
    ```

    The `-T` flag sends the file using an HTTP `PUT` request. Dufs creates any missing directories in the path automatically.
  </Step>
  <Step title="Download a file">
    ```sh
    curl http://127.0.0.1:5000/path-to-file
    ```

    To download a whole folder as a zip archive:

    ```sh
    curl -o path-to-folder.zip http://127.0.0.1:5000/path-to-folder?zip
    ```
  </Step>
  <Step title="Resumable downloads">
    ```sh
    curl -C- -o file http://127.0.0.1:5000/file
    ```

    The `-C-` flag tells curl to resume an interrupted download automatically using HTTP range requests.
  </Step>
</Steps>

<Tip>Both HTTP Basic and Digest auth are supported. Pass credentials with `--user user:pass` or `--user user:pass --digest`.</Tip>

## Health check

Use the built-in health endpoint to verify the server is running — useful in scripts, Docker health checks, and CI pipelines:

```sh
curl http://127.0.0.1:5000/__dufs__/health
```

A `200 OK` response confirms the server is up.

## Common one-liners

<CodeGroup>

```bash Serve current directory (read-only)
dufs
```

```bash Enable all operations
dufs -A
```

```bash Serve a specific directory
dufs Downloads
```

```bash Require a username and password
dufs -a admin:123@/:rw
```

```bash Listen on a custom host and port
dufs -b 127.0.0.1 -p 8080
```

```bash Serve over HTTPS
dufs --tls-cert my.crt --tls-key my.key
```

</CodeGroup>
